1. OBJETIVO DO PROJETO
----------------------
Sistema RAG (Retrieval-Augmented Generation) para indexar READMEs de projetos
da empresa e permitir busca semântica para verificar se determinada ferramenta
ou funcionalidade já existe em algum repositório no GitLab.


2. O QUE FOI FEITO
-------------------

2.1 Migração ChromaDB → PostgreSQL + pgvector
    - Removido ChromaDB como banco vetorial
    - Adicionado PostgreSQL 16 com extensão pgvector
    - O banco roda via Docker (docker-compose.yml)
    - Índice HNSW com operação de distância cosseno para busca rápida
    - Tabela "chunks" armazena: id, repo_name, content, metadata, embedding

2.2 Docker
    - Criado docker-compose.yml com imagem pgvector/pgvector:pg16
    - Container nomeado "synapse_pgvector"
    - Volume persistente "pgdata" para não perder dados ao reiniciar
    - Healthcheck configurado com pg_isready

2.3 CLI (cli.py)
    - Criada interface de linha de comando usando argparse + rich
    - 3 comandos: index, search, stats
    - Atalho: se o primeiro argumento não for um comando, assume "search"
    - Aceita busca direta: python cli.py "minha consulta"
    - Saída formatada com tabelas coloridas (rich)

2.4 20 READMEs Fictícios
    - Criados 20 arquivos .md na pasta readmes/ para teste
    - Todos em português, simulando projetos reais de uma empresa
    - Domínios: autenticação, pagamentos, chatbot, CI/CD, estoque, etc.

2.5 API FastAPI (mantida)
    - API REST continua funcional em /api/v1
    - Endpoints: POST /index, DELETE /index/{repo}, POST /search, GET /stats
    - testar_readme.py testa via HTTP (precisa do servidor rodando)
